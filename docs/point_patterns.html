5 Point Patterns

5.1 Introduction

Aims

The aims of this practical are to:

  1. Understand the key differences between point-pattern analysis and point-process modelling.
  2. Understand common approaches used to analyse the spatial distribution of point data.
  3. Apply these approaches to an ecological dataset.

Application

To that end, we’ll analyse the patterns in a Point dataset, using research-grade species observations of Panthera tigris from iNaturalist, obtained via the Global Biodiversity Information Facility (GBIF).

Data

The data for this practical are available in [data/gbif] as follows:

  • Species observations of Panthera tigris (N=4449, 2 June 2026) obtained via GBIF.org [Source]
  • Protected areas of India (National Parks and Wildlife Sanctuaries), built from OpenStreetMap data [Source]

Tools

Point Patterns

Kernel Density (Spatial Analyst Tools), Mean Center (Spatial Statistics Tools), Median Center (Spatial Statistics Tools), Standard Distance (Spatial Statistics Tools), Directional Distribution (Standard Deviational Ellipse) (Spatial Statistics Tools), Average Nearest Neighbor (Spatial Statistics), Multi-Distance Spatial Cluster Analysis (Ripley’s K Function) (Spatial Statistics), Find Point Clusters (GeoAnalytics Server Tools)

Data Management

Spatial Join (Analysis Tools), Clip (Analysis Tools), Dissolve (Data Management Tools), Generate Tessellation (Data Management Tools), Minimum Bounding Geometry (Data Management), Create Random Points (Data Management), XY Table To Point (Data Management Tools), Calculate Geometry Attributes (Data Management Tools)

To do? Remove non-contiguous part? i.e., Andaman Islands, Maldives. This could be achieved using Multipart To Singlepart (Data Management Tools)

[5] Update Projection

In the previous practicals, we’ve largely been dealing with topological issues (i.e., neighbouring, bordering) and haven’t deal in great depth with geometric issues i.e., areas, distances (link to Understanding GIS, Week 2)

In this practical, we are certainly in the realm of geometry i.e., are points clustered or randomly dispersed? As a result, the choice of projection is an important one!

How do we select a suitable CRS?

  • Best option (for global analysis or if in doubt) = ellipsoidal / geodesic measurement
  • Smaller areas, projected CRS, selecting a property to preserve (conformal, equal area, equidistant)
  • For individual countries, we can often use the national grid e.g., the CRS for the UK is British National Grid (conformal), but is sufficiently accurate for area and distance calculations

For our study area, which doesn’t fit neatly into a national grid, we can select via:

  • The Projection Wizard:
    • Select area of interest and distortion property
    • Open WKT information
    • Copy to a text file and save with the extension .prj, denoting a projection file
    • In ArcGIS, we can apply this to our Map using Properties -> Coordinate Systems -> Add, then Import Coordinate System and Apply.
  • Alternatively, we can let ArcGIS suggest one for us:
    • New suggested projected coordinate system
    • Name and choose GCS (WGS 1984 - link to Understanding GIS here)
    • Extent of a layer (Natural Earth data subset to Indian subcontinent)
    • Select distortion Property (conformal, equal-area, etc) and Save

Use either method to select a suitable conformal projection for our area of interest.

Note this applies to the map visualisation, but doesn’t change the underlying data in any way, which are reprojected “on-the-fly`.

To change the feature data, use Project on the Point data.

5.2.2 Visualisation and Centrography

[6] In our first step, we are going to create a grid across the study area, and then symbolise the cells based on the # species observations. This is useful, because when looking at the points in isolation (at the sub-continent scale), there are lots of high observation clusters. When tesselated, there are one or two areas that stand out (>500 observations).

We could also use this as the basis for statistical testing e.g., \(χ^2\), where we could compare the # observations in each cell or quadrat vs. an even distribution. However, this is rather tricky to do in ArcGIS!

Run Generate Tessellation:

  • Extent = Natural Earth data subset to Indian subcontinent
  • Shape Type = Square
  • Size = 2000 km^2
  • Proj = Match extent layer

[7] Next, we will use a Spatial Join:

  • Target = Output of Generate Tessellation,
  • Join = occurrence points
  • Field name = Count
  • Join Operation = one to one
  • Match Option = Completely contains [or intersects])

Discuss Symbology effects, Graduated Colours (using quantiles etc) or Unclassed Colours (min to max)

[8] Run Kernel Density:

  • Input = occurrence points
  • Population field = NONE
  • Cell size e.g., 10_000 m
  • Method = Geodesic (incorporating Earth shape)

Explore changing search radius e.g., 20,000 m2 to 200,000 m2 Explore effects of barriers (e.g., Natural Earth polygon).

Note this tool produces values within the data range (min-max xy) and creates static layers.

We can use a dynamic approach via Symbology, which is more flexible, but arguably less robust i.e., Radius is in screen units (points), rather than map units (meters, feet, etc)

Explore constant vs. dynamic approach.

[9] Run Mean Center and Median Center

  • Provides information on the center of Mass e.g., discrepancy of mean-median caused by skew. For the median, 50% of points above+below point, and left+right. For the tiger dataset, the mean is skewed north by Himalayan observations

[10] Run Standard Distance i.e., on average, how far are observations from the geometric mean center?

  • For the occurrence dataset, this produces 559904.230305 m i.e., on average, observations are ~560 km from the mean center. The reported distance is in the units of the projection here. In [5] we switched to a projected CRS, with units in meters, but if we were to use a geographic projection, units would typically be in degrees.

[11] Run Directional Distribution (Standard Deviational Ellipse), which provides information on the center, dispersion, and orientation.

  • For the occurrence dataset, CenterX and CentreY should be identical to outputs of mean centre and standard distance tools
  • The shape of the output indicates wider disperson along N-S axis (YStdDist) vs. E-W axis (XStdDist), with rotation of ~12deg.
  • This probably reflects outlier points in Bangladesh, Assam etc, coupled with points in SW India

5.2.3 Randomness

[12] Run Average Nearest Neighbor, which is Clark-Evans R (Clark and Evans, 1954).

  • Use Euclidean distances (projected data)
  • Generate report (html)
  • Area is critical - the default is the minimum bounding rectangle enclosing all of the observations

Default Output:

  • Observed Mean Distance = 1830.375410
  • Expected Mean Distance = 16414.771642
  • Nearest Neighbor Ratio = 0.111508
  • z-score = -109.656790
  • p-value = 0.000000
  • Given the z-score of -109.65679, there is a less than 1% likelihood that this clustered pattern could be the result of random chance.

Very straightforward implementation and interpretation, but key discussion points:

Discuss:

  • Analytical approach
  • Distance calculations (observed and expected mean distance)
  • Nearest neighbor ratio
  • Z-scores
  • Introduce the Observation Window

Evaluate the impact of changing the observation window (area), for example using the area of the Natural Earth polygons, calculated using Calculate Geometry Attributes

5.2.4 Cluster analysis

[13] Run Multi-Distance Spatial Cluster Analysis (Ripley’s K Function), developed by Ripley (1988).

  • This tool is very slow with high number of permutations, and very slow even with no permutations
  • As above, it is highly sensitive to “study area” method, as described here
  • Boundary correction is also important.

Discuss the Ripley alphabet of functions (\(G, F, K\))

Interpretation:

  • Observed K [actual data] > Expected K [complete spatial randomness] = Clustered i.e., points closer together than by pure chance, at that specific distance
  • Observed K < Expected K = Dispered i.e., points spread out compared to a random pattern
  • Visually, when Observed K line far from Expected K line, clustering is statistically significant

[13] Run Find Point Clusters, key methods include DBSCAN (defined-distance) and HDBSCAN (self-adjusting)

Important choices include the minimum number of features (% of dataset) and search distance. This flexibility is a strength and a limitation, as the # and distribution of clusters is highly sensitive to user choice.

Reasonable selection of parameters:

  • Search distance based on ecological grounds e.g., an average home range of 110 km2 for Sundarban of India and Bangladesh (Naha et al., 2016), although they can travel much further (Hussain et al., 2022)!
  • Minimum number - 45 (1%)
  • This generate a reasonable pattern (7 main clusters, plus outliers)

HDBSCAN gives different results, varying distance based on point density (central block broken up into smaller clusters, and v.v elsewhere). For this dataset, DBSCAN makes more sense, using a fixed distance, based on ecological evidence (rather than the degree of Point clustering)

To finish the practical:

  • Ensure the practical finishes with a natural progression to point-process modelling i.e., all of this is great, and tells us about the pattern of points, but not how that pattern arises. This is outside the scope of this unit, but you can explorer ecological examples in Spatial Ecology in Semester 2.
  • Emphasise the importance of the Observation Window.
  • Touch on uncertainty, with many points at the same location (n=125), reported positional uncertainty of ±30 km. How important is this at our scale of analysis?
  • DON’T discuss presence-absence problems here, as this will be better suited to the interpolation practical.

Downloading data from GBIF

To download species-observation data manually via GBIF:

  • Register for GBIF
  • Get data - Species
  • Full text search e.g., Canis lupus (Linnaeus, 1758, Gray Wolf)
  • Select Occurrences
  • Apply Filters of interest e.g.,
    • Country or area (United States of America)
    • Dataset (iNaturalist Research-grade Observations)
    • Location (Has coordinates = Yes”)
  • Download simple occurrence data (csv)
  • If using and publishing the data, include the recommended data attribution e.g., GBIF.org (7 July 2026) GBIF Occurrence Download https://doi.org/10.15468/dl.vdnczk

It is also possible to download data via a geoprocessing tool in ArcGIS [Source] [Instuctions].
