5 Point Patterns

5.1 Introduction

Aims

The aims of this practical are to:

  1. Understand the key differences between point-pattern analysis and point-process modelling.
  2. Understand common approaches used to analyse the spatial distribution of point data.
  3. Apply these approaches to an ecological dataset.

Application

To that end, we’ll analyse the patterns in a Point dataset, using research-grade species observations of Panthera tigris from iNaturalist, obtained via the Global Biodiversity Information Facility (GBIF).

Data

The data for this practical are available in [data/gbif] as follows:

  • Species observations of Panthera tigris (N=4449) obtained via GBIF [Source]
  • Protected areas of India (National Parks and Wildlife Sanctuaries), built from OpenStreetMap data [Source]

Tools

Point Patterns

Kernel Density (Spatial Analyst Tools), Mean Center (Spatial Statistics Tools), Median Center (Spatial Statistics Tools), Standard Distance (Spatial Statistics Tools), Directional Distribution (Standard Deviational Ellipse) (Spatial Statistics Tools), Average Nearest Neighbor (Spatial Statistics), Multi-Distance Spatial Cluster Analysis (Ripley’s K Function) (Spatial Statistics), Find Point Clusters (GeoAnalytics Server Tools)

Data Management

Spatial Join (Analysis Tools), Clip (Analysis Tools), Generate Tessellation (Data Management Tools), Minimum Bounding Geometry (Data Management), Create Random Points (Data Management), XY Table To Point (Data Management Tools)

[5] Projection

  • In the previous practicals, we’ve largely been dealing with topological issues (i.e., neighbouring, bordering) and haven’t therefore had to deal with geometric issues i.e., areas, distances (link to Understanding GIS, Week 2)
  • In this practical, we are considering those issues i.e., are points clustered or randomly dispersed?
  • As a result, the choice of projection is an important one!
  • How do we select:
    • Best option (for global analysis or if in doubt) = ellipsoidal / geodesic measurement
    • Smaller areas, projected CRS, selecting a property to preserve (conformal, equal area, equidistant)
    • For individual countries, we can often use the national grid e.g., the CRS is British National Grid (conformal), but is sufficiently accurate for area and distance calculations
  • For area, we can select via:
    • The Projection Wizard -> Copy WKT to text, save as .prj -> Import Coordinate System
    • New suggested projected coordinate system -> Extent of a layer -> Property (conformal, equal-area, etc)

[12] Multi-Distance Spatial Cluster Analysis (Ripley’s K)

-> Very slow with high number of permutations, and very slow even with no permutations -> Highly sensitive to “study area” method, see: https://doc.esri.com/en/arcgis-pro/latest/tool-reference/spatial-statistics/multi-distance-spatial-cluster-analysis.html?tabs=dialog

[TO HIGHLIGHT] Observation Window [TO INVESTIGATE] Boundary correction [UNDERSTAND] The Ripley family - how is the ‘K’ in ArcGIS different from G and F in “Geographic Data Science with Python”

This is a reference to Ripley (1988).

Interpretation: - Observed K [actual data] > Expected K [complete spatial randomness] (Clustered) - points closer together than by pure chance, at that specific distance - Else Observed K < Expected K (Dispered) - points spread out compared to a random pattern

Visual: - Observed K line far from Expected K line = clustering is statistically significant

[13] Find Point Clusters (GeoAnalytics)

-> Key methods: DBSCAN (defined-distance) and HDBSCAN (self-adjusting)

[KEY CHOICES] Minimum number of features (% of dataset), and search distance - This flexibility is a strength and a limitation - Search distance - we should do this based on ecological grounds! (~100 km2) - Minimum number - 45 (1%) - This gives a reasonable pattern (7 main clusters, plus outliers)

-> HDBSCAN gives different results, varying distance based on point density (central block broken up into smaller clusters, and v.v elsewhere) -> For this data, DBSCAN makes more sense - fixed distance, based on ecological evidence (not Point clustering)

—————– Ripleys (AI overview)

  1. Ripley’s K (Multi-Distance Spatial Cluster Analysis)What it does: It counts the number of neighboring features found at various distances from each feature.Use case: It helps you find if your points are clustered, dispersed, or random at different scales (e.g., are crime hotspots tight and localized, or spread out across a whole city?).ArcGIS Tool: Multi-Distance Spatial Cluster Analysis (Spatial Statistics toolbox).

  2. Ripley’s G (Nearest Neighbor Event-to-Event)What it does: It calculates the distance from each point to its nearest neighboring point and plots the cumulative distribution of these distances.Use case: It is perfect for testing if specific features (like specific plant species or retail store locations) are clustered together. If \(G(d)\) shows 80% of points have a nearest neighbor within 10 meters, it indicates tight clustering.ArcGIS Tool: Average Nearest Neighbor or evaluated using Spatial Autocorrelation.

  3. Ripley’s F (Empty Space or Point-to-Event)What it does: Instead of measuring from feature to feature, it measures the distance from a set of randomly placed points (empty spaces) to the nearest feature.Use case: It measures the “gaps” between features. For example, it can help determine if a city has an even distribution of parks by measuring how far a randomly selected residential street is from the nearest green space
