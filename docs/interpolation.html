6 Interpolation

6.1 Introduction

Aims

The aims of this practical are to:

  1. Introduce geostatistical approaches for analysing spatial data.
  2. Understand how common spatial interpolation techniques work.
  3. Perform interpolation in ArcGIS and become familiar with common applications e.g., for prediction, as an input for spatial regression.

Application

To achieve this, we’ll apply interpolation techniques to Point samples of global plant trait data (leaf area, plant height), modelled by Lusk et al. (2026). Our study area is Peru, which features extreme gradients in climate and topography, which are likely to be reflected in spatial gradients in plant traits.

Data

This practical uses global plant trait data derived from crowdsourced biodiversity monitoring and Earth observation, available in [data/global-plant-traits] [Paper] [Source]

Tools

Extract Values to Points (Spatial Analyst Tools), IDW (Geostatistical Analyst Tools), Empirical Bayesian Kriging (Geostatistical Analyst Tools), Geostatistical Wizard, Exploratory Interpolation (Geostatistical Analyst Tools)

Interpolation techniques are used to estimate unknown values at new locations based on known measurements from surrounding sample points.

We could use our species observation Points (Panthera tigris) from Practical 5 here, but it’s not an ideal example, for a few reasons:

  1. Our Points don’t have corresponding values that could be used for spatial interpolation e.g., animal height, age, or mass for each observation.

  2. As our Points represent individual observations, we would typically have to aggregate these data in some way to perform interpolation i.e., either via tessellation to a grid or by generating a new Point dataset, where the value represents the number of observations within a specified distance.

  3. As demonstrated in Practical 5, the distribution of Panthera tigris observations is highly clustered and non-continuous, with observations primarily in designated nature reserves. Interpolation techniques will typically struggle to capture this spatial pattern, and instead, we’d probably want to use a species distribution model (an example of a point-process model) in this situation. You can learn about these in Spatial Ecology in Semester 2.

  4. Finally, our species observations are Presence-Only data (i.e., recorded observations of Panthera tigris), and we have no access to corresponding Absence-data, which you can read about more fully in Grattarola et al. (2023) and Kent and Carmel (2011). As a result, we do not know whether the “gaps” in our Point-pattern analysis reflects true absence, or simply sampling bias (Hughes et al., 2021).

Instead, we are going to apply interpolation to a dataset where we know the spatial pattern is continuous, and where we have a variable we want to predict.

We can also change ‘Sector Type’. Default = 1 i.e., we will look for nearest N neighbours within that area. If we increase to 4 or 8 sectors (and modify their Angle), we look for N neighbours per sector.

An important setting is power which determines the influence of the input points on the prediction, as a function of distance. The default is 2 i.e., \(distance^2\). As we increase this parameter, closer points play an increasingly important role in the output. Compare power = 1 - 10.

Power can also be optimised, which is via root-mean-square cross validation which we’ll discuss more fully in the next section.

[11] Cross validation of IDW

On the next tab, we are presenting with predicted values, errors, and the distribution of the predicted values.

These are derived from cross-validation, which involves removing a single point from the input dataset, re-running interpolation, and then evaluating the deviation of the predicted value from the input value. This is repeated for all points in the input dataset.

This can be used to calculated root-mean-square error (RMSE):

\[\sqrt{\sum_{i=1}^n(O_i-M_i)^2} \] which is the square root of the sum (\(\sum\)) of the squared differences between observed values (\(O_i\), input specific leaf area) and modelled values (\(M_i\), interpolated specific leaf area). This is the average distance between the model predictions and the observed values and is expressed in the units of the observed values (i.e., SLA).

6.2.4 Geostatistical interpolation

To start, we’ll discussing the origin of geostatistics and the key differences with deterministic methods i.e., incorporating the spatial structure of the data (variography, spatial autocorrelation). As summarised in the documentation: “Some of the spatial variation … can be modeled by random processes with spatial autocorrelation, and require that the spatial autocorrelation be explicitly modeled.”

Kriging, developed and popularised by Krige (1951) and Matheron (1963), is synonymous with the field of geostatistics, and comes in many versions, including Ordinary, Universal, Co-kriging, Indicator, and Empirical Bayesian, some of which we’ll use today. Understanding how these work, and selecting the appropriate version for your data and analysis, is therefore critical.

6.3 Extra

In the practical we’ve used cross-validation to assess interpolation quality. We can also assess the quality of the interpolation using an independent set of validation locations. This can be achieved by:

  • Generating a set of validation locations via Create Random Points
  • Extracting the corresponding values from the input data and the interpolated layers using Extract Multi Values to Points
  • Create Chart - Scatter Plot - R2

This approach is only possible because our analysis is based on a continuous input surface [3117-leaf-area-peru.tif]. Typically, interpolation of Point data would be used to create a continuous surface - if we have it already, we wouldn’t need to interpolate!

We also need to evalute performance cautiously, acknowledging that the input is also a model.
