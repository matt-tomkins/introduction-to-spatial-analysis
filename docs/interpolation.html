6 Interpolation

6.1 Introduction

Aims

The aims of this practical are to:

  1. Introduce geostatistical approaches for analysing spatial data.
  2. Understand how common spatial interpolation techniques work.
  3. Perform interpolation in ArcGIS and become familiar with common applications e.g., for prediction, as an input for spatial regression.

Application

To achieve this, we’ll apply interpolation techniques to Point samples of global plant trait data (leaf area, plant height), modelled by Lusk et al. (2026). Our study area is Peru, which features extreme gradients in climate and topography, which are likely to be reflected in spatial gradients in plant traits.

Data

This practical uses global plant trait data derived from crowdsourced biodiversity monitoring and Earth observation, available in [data/global-plant-traits] [Paper] [Source]

Tools

Extract Values to Points (Spatial Analyst Tools), IDW (Geostatistical Analyst Tools), Empirical Bayesian Kriging (Geostatistical Analyst Tools), Geostatistical Wizard, Exploratory Interpolation (Geostatistical Analyst Tools)

6.2 Practical

Interpolation techniques are used to estimate unknown values at new locations based on known measurements from surrounding sample points.

We could use our ecological dataset from Practical 5 here, but it’s not an ideal example, for a few reasons:

[1] Our Points don’t have corresponding values that we would want to predict i.e., characteristics of each observation [2] Our Points represent individual observations, so to interpolate, we would have to aggregate these in some way i.e., # observations with a certain distance of some representative locations (schematic?) [3] Fundamentally, the distribution of Tiger observations is clearly non-continuous, found mostly in designated reserves. Interpolation will probably do a poor job at capturing this spatial pattern, and instead, we’d probably want to use a species distribution model (Link to Spatial Ecology) [4] As a final reason, our data are Presence-Only data (i.e., observations of Tigers), and we have no Absence-Data, see: - https://doi.org/10.1111/jbi.14622 - https://doi.org/10.1111/j.1472-4642.2011.00755.x

Instead, we are going to apply interpolation to a dataset where we assume the spatial pattern is continuous, and where we have a variable we want to predict.

By comparing the interpolation output (e.g., Universal Kriging) with the “true” values from the input data i.e., residual.

Need to do so cautiously, acknowledging that the input is also a model.

However, this is important for validation of interpolation outputs - how well does our prediction match real empirical measurements?

Fundamentally, interpolation is important when we are studying something that will expected to vary continuously through space (even if it goes to 0) e.g., soil depth, elevation, species richness, but only have measurements of that variable at discrete locations.
