Introduction to Spatial Analysis

Chapter 2 Spatial Weights

2.1 Introduction

Aims

The aims of this practical are to:

  1. Understand what spatial weights represent.
  2. Understand how spatial weights are constructed (i.e., contiguity, distance).
  3. Understand how spatial weights underpin analysis of spatial patterns.

Application

To achieve this, we’ll demonstrate construction of spatial weights using country Polygons and assess neighbourhood effects on cultural similarity, using the Cultural Fixation Index (CFST) from Muthukrishna et al. (2020).

Data

  • Natural Earth 1:10m Cultural Vectors, available in [data/natural-earth] [Source]
  • Cultural Similarity matrix, produced for 80 country pairs by Muthukrishna et al. (2020), available in [data/cultural-similarity] [Source]

[Optional] Use OS 100 km grid as demonstration… [Source] but needs sorting (64-bit shapefile issue)

Tools

Generate Spatial Weights Matrix (Spatial Statistics), Convert Spatial Weights Matrix to Table (Spatial Statistics), Introduction to Neighborhood Explorer, Add Geometry Attributes (Data Management Tools), Join Field (Data Management Tools), XY To Line (Data Management Tools)

2.2 Practical

During this practical…

[1] Open ArcGIS Pro

[2] Load vector data (Natural Earth)

[3] Run Generate Spatial Weights Matrix - talk through different settings, using a unique ID (NE_ID)

[4] Use Neighborhood Explorer to visualise, we can either do this dynamically or use the .swm file created earlier — the former is the best approach for the students to quickly see the differences

[5] This allows us to visualise the connections for a single feature. To visualise all relationships, we need to use Convert Spatial Weights Matrix to Table (Spatial Statistics)

———Rather than NE_ID, shall we change to ADMIN?

[6] This table contains the IDs of the start countries (NE_ID) and the end countries (NID) i.e., UK = 78, Ireland = 77, but no geometry

[7] So, run Add Geometry Attributes (Data Management Tools) to Natural Earth data (centroid x and y)

[8] Then use Join Field based on ID > Input = swm > Input field = NE_ID > Join table = natural_earth > Join field = NE_ID > Use Field Mapping - Remove all but centroid_x and centroid_y > Double click to rename these as source_x and source_y, validate and run

Then repeat for the target locations, which is as simple as changing the input file = NID, and to target_x and target_y

[9] Now we can use [XY to Line], using source_xy and target_xy, “and preserve attributes”

[10] To link the lines to cultural similarity, we need to include the country names (‘ADMIN’) > Join Field again, same process, with name as source_ADMIN > Repeat using NID as input field, updating to target_ADMIN

[10?] Critical engagement with France?
