2 Spatial Weights

2.1 Introduction

Aims

The aims of this practical are to:

  1. Understand what spatial weights represent.
  2. Understand how spatial weights are constructed (i.e., contiguity, distance).
  3. Understand how spatial weights underpin analysis of spatial patterns.

Application

To achieve this, we’ll demonstrate construction of spatial weights using country Polygons and assess neighbourhood effects on cultural similarity, using the Cultural Fixation Index (CFST) from Muthukrishna et al. (2020).

Data

  • Natural Earth 1:10m Cultural Vectors, available in [data/natural-earth] [Source]
  • Cultural Distance matrix, produced for 80 country pairs by Muthukrishna et al. (2020), available in [data/cultural-similarity] [Source]
  • Ordnance Survey (OS) 100 km grid, available in [data/os/grids] [Source]

Tools

Generate Spatial Weights Matrix (Spatial Statistics), Convert Spatial Weights Matrix to Table (Spatial Statistics), Introduction to Neighborhood Explorer, Add Geometry Attributes (Data Management Tools), Join Field (Data Management Tools), XY To Line (Data Management Tools), Delete Identical (Data Management Tools), Summary Statistics (Analysis Tools), Create Thiessen Polygons (Analysis Tools)

2.2 Practical

To begin, a conceptual discussion of spatial weights.

2.2.1 Spatial weights using a grid

[1] Open ArcGIS Pro

[2] Load OS grid data [osbng_grid_100km.shp]. This is a very simple dataset, so easy to see the effects of the various neighbourhood settings, before moving on to a more complex dataset

[3] Use Neighbourhood Explorer to visualise different spatial weights settings.

To discuss, neighbourhood types (distance-based, contiguity)

2.2.2 Spatial weights using country Polygons

[4] Load country data [ne_10m_admin_0_countries.shp] and repeat process.

[5] The above allows us to dynamically view connections for a single feature of interest (Select Focal Feature). To generate these connections for the entire dataset, we can use Generate Spatial Weights Matrix - talk through different settings, using a unique ID (NE_ID)

[6] Return to the Neighbourhood Explorer, where we can now use our created swm file as an input.

[7] To visualise all relationships, we need to use Convert Spatial Weights Matrix to Table (Spatial Statistics)

[8] The resulting table contains the IDs of the start countries (NE_ID) and the end countries (NID) i.e., \(UK = 78\), \(Ireland = 77\), but no corresponding geometry

[9] Run Add Geometry Attributes (Data Management Tools) to Natural Earth data (centroid x and y)

[10] Then use Join Field based on ID:

  • Input = swm
  • Input Field = NE_ID
  • Join table = natural_earth
  • Join field = NE_ID
  • Use Field Mapping:
    • Remove all but centroid_x, centroid_y and ADMIN
    • Double click to rename these as source_x, source_y, source_admin
  • Validate and run

Note we are including the ADMIN attribute here, as we will use this later to link to cultural distance index

[11] Then repeat for the target locations, which is as simple as changing the input file = NID, and to target_x, target_y and target_admin

[12] Now we can use XY To Line, using source_xy and target_xy, “and preserve attributes” (all other settings as default)

[13] Engage critically with visualisation e.g., France? (centroid location is outside of the contiguous part of France, due to S. America)

2.2.3 Comparison with cultural distance

[14] Open the matrix of cultural distance [cultural-distance.csv] which I’ve edited to leave only unique combinations (using “-”.join(sorted([!country_1!, !country_2!]))), as summarised in join_key. Our task is to match those to the data created above i.e.,

Are ‘neighbouring’ countries (as defined by the swm), more culturally similar?

Note the smaller the cultural distance value, the more similiar the countries are e.g., Egypt and Norway are dissimiliar (0.5953), whereas Australia and the UK are highly similar (0.0319)

[15] This csv is not editable so Export Table, and save in a gdb i.e., practical-2.gdb. Remove the original csv from ArcGIS to avoid confusion

To do, discuss geodatabases?

[16] Next use Delete Identical, with the XY to Line layer created in [12], with Fields = Shape, to remove comparable geometries e.g., (IRE-UK, UK-IRE, N=344)

[17] Create a target_key in the XY to Lines dataset via Attribute Table -> Calculate Field. Equation: “-”.join(sorted([!Source_ADM!, !Target_ADM!]))

[18] Using Join Field, with the editable cultural distance (gdb) dataset as input:

  • Input Field = join_key (created by MT)
  • Join Table = cultural-distance
  • Join Field = target_key (created in [17])
  • Transfer Fields = target_key

Note if there is a match, the target_key field will have data, else <Null>

[19] Calculate Field (distance_table): neighbours (text): “false” if !target_key! is None else “true”

Note I originally had this as “Non-bordering” and “Bordering” but this could be misleading. This is only correct if a contiguity-based SWM was created, and not a distance-based rule

[20] Finally, use [Summary Statistics] to outline results:

  • Statistic Fields = distance [Mean, SD]
  • Case Fields = neighbours

2.2.4 Results

[21] Inspect output of Summary Statistics

  • Neighbouring country distance mean = 0.07, SD = 0.05 (n=85)

Note This is for where countries are neighbouring (as defined by the swm) AND there is a cultural distance value i.e., UK-Ireland not in the dataset.

  • Non-neighbouring country distance mean = 0.14, SD = 0.08 (n=3075)

[22] Interpretation: non-neighbouring countries are characterised (on average) by greater cultural distance (0.14 vs. 0.07, 2x). However, we can be critical here:

  • Extreme un-eveness of sample sizes
  • Overlap with SD (t-test for significance?)
  • Sensitive to our designation of neighbourhood-ness
  • Sensitive to calculation of cultural similarity/distance. Can this be expressed well with a single value?
